基本思路：
1. 我想做一个Autoskills的MCP服务，每当agent解决一个问题时/ 成功完成一个task时，就自动回顾判断一下当前我们的solution是否值得记录成一个值得复用的agent skill。
2. 如果在解决这个问题的过程当中没有用到任何skill，且你觉得我们的solution是值得打包成为一个agent skill的，就在对话中提问user，是否需要打包成一个skill并简短的说明理由。
3. 如果在这个任务完成的过程当中已经使用了某个或某些skill，可以复盘一下我们的任务过程，如果这个skill看起来很顺畅的完成了它的职责，就不触发提问，如果这个skill看起来没有成功适配我们的任务，执行过程不流畅不成功，则提问user是否需要改进这个agent skill。
4. 这个MCP服务需要适配 kilo code / Claude code / cursor / windsurf 等agent应用，同时这类由我们的autoskills的MCP打造的skills库，可以统一放进一个独立的personal skills文件夹（但是得保证可以被读取执行），因为user肯定希望有些skills是冻结的，而只有personal skill是需要被更新以及添加创建的，我们的personal skills库可以方便我们复用到之后类似的任务当中。
5. 将你所有的脚本和需要的内容全部都放置在我们当前工作目录下，方便我开源成一个Github仓库，同时别人也可以很方便的使用我们的Autoskills MCP。

创建skills的重要步骤：
  步骤 1: 创建 Skill 模板
  npx skills init <skill-name>
  npx skills init 会生成带有模板的 SKILL.md 文件，在当前工作目录生成 <skill-name>/SKILL.md。
  
  步骤 2: 编辑 Skill 定义
  编辑 SKILL.md，主要修改：
  ┌──────────────┬────────────────────────┐
  │     部分     │          说明          │
  ├──────────────┼────────────────────────┤
  │ name         │ skill 的唯一标识符     │
  ├──────────────┼────────────────────────┤
  │ description  │ 简短描述，用于匹配触发 │
  ├──────────────┼────────────────────────┤
  │ 标题         │ 显示名称               │
  ├──────────────┼────────────────────────┤
  │ When to use  │ 触发条件说明           │
  ├──────────────┼────────────────────────┤
  │ Instructions │ 具体工作流程           │
  └──────────────┴────────────────────────┘

  步骤 3: 安装到本地所有agent应用
  npx skills add ./<skill-name>
